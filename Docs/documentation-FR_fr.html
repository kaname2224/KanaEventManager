Description

KanaEventManager est un add-on à KanaBungeeTP qui se chargera de gérer vos évènement de serveur de façon rapide et intuitive grâce à des commandes précises tout cela en interserveur

Pour utiliser KanaEventManager vous devez avoir KanaBungeeTP installé et configuré Voir les téléchargements
Ainsi qu'une base de données fonctionnelle

Télechargement

Vous devez avoir KanaBungeeTP, téléchargez le sur SpigotMC

Installation

- Installez et configurez KanaBungeeTP
- Eteignez votre serveur et mettez KanaEventManager dans le dossier plugin de votre serveur d'évènement
- Redemarrez votre serveur et vérifiez que KanaBungeeTP et KanaEventManager démarrent
- Ouvrez le config.yml de KanaEventManager et rentrez vos informations de connexion à la base de données
- Entrez le nom des serveurs lobby et event (ces nom doivent être ceux définie dans votre configuration BungeeCord)
- Voilà KanaEventManager est configuré et prèt à l'emplois !

Commandes

Nous pouvons donc passez aux différente commandes du plugins
Les mots en majuscule dans le nom de la commandes sont les paramètres pris en comptes

/manageEvent => Commande de base affiche cette documentation (Alias : /event, /e)

/manageEvent create NOM BROADCAST => Permet de créer un évènement elle prend en parametre un nom et un broadcast

/manageEvent start NOM -p|-t VALEUR => Prépare un event et ouvre le serveur event au public
-p : permet de définir un nombre de joueur maximum des que ce nombre est atteint le serveur se fermera au public
-t : permet de mettre un temps en secondes pendant lequels le serveur event sera ouvert au public

/manageEvent infos => Donne des informations sur l'event en cours ou en préparation
tel que son createur, le nombre de participants, son status actuel

/manageEvent listPlayer => Donne la liste des pseudos des participants

/manageEvent launch => Lance l'event préparé si celui-ci est prêt

/manageEvent stop => Arrête l'event et éjecte les participants

/manageEvent leave => Permet de retourner au lobby

/manageEvent forceReady => Permet de lancer un event sans attendre la fin du temps imparti ou le nombre de joueurs requis
